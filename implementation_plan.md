# 카드 거래 중계 플랫폼(Card Trading Platform) 개발 계획서

본 플랫폼은 **SNKRDUNK**를 벤치마킹하여, 희귀 카드(포켓몬, 유희왕 등)의 거래를 보다 안전하고 투명하게 중계하는 **도감 기반 출품형 카드 마켓플레이스**입니다.

플랫폼은 카드 도감 정보를 체계적으로 제공하고, 회원이 해당 카드에 자신의 실물 카드를 출품하여 판매할 수 있도록 지원합니다. 또한 거래 과정에서 **에스크로 기반 결제 보관, 정산, 분쟁 신고 접수 및 정책 범위 내 중재**를 통해 거래 안정성을 높이는 것을 목표로 합니다.

본 플랫폼은 **SNKRDUNK의 카드 도감/시세/출품 구조**와 **당근마켓·번개장터의 개인 간 거래 UX**를 참고하여 설계합니다.

---

## 1. 서비스 개요

본 서비스는 단순한 중고 거래 게시판이 아니라, **카드 도감(CardMaster)** 을 중심으로 거래가 이루어지는 구조를 가집니다.

- 관리자는 거래가 활발한 카드들을 도감 형태로 등록 및 관리합니다.
- 회원은 등록되지 않은 카드에 대해 카드 추가를 요청할 수 있습니다.
- 회원은 도감에 등록된 카드에 대해 자신의 실물 카드를 출품할 수 있습니다.
- 구매자는 카드 상세 페이지에서 출품 목록, 최저가, 최근 거래 이력, 상태별 시세 등을 확인한 뒤 구매할 수 있습니다.
- 각 회원은 공개 프로필 및 상점 페이지를 가지며, 판매 목록과 거래 후기, 평점을 노출합니다.
- 플랫폼은 카드 감정 기관이나 물류 대행사가 아니며, **결제 보관, 정산, 분쟁 신고 접수, 운영정책 범위 내 중재**를 담당합니다.

## 2. 시스템 아키텍처

- **백엔드**: Spring Boot (Java 17+), Gradle 기반 RESTful API 서버
- **프론트엔드**: Vue.js 기반 독립 클라이언트
- **데이터베이스**: MySQL 8.0+
- **ORM/Data Access**
    - Spring Data JPA: 일반 CRUD 및 엔티티 매핑
    - MyBatis: 복잡한 검색, 필터링, 시세/거래 이력 조회 최적화
- **캐시**: Redis
    - 카드별 최저가
    - 최근 거래가
    - 인기 카드
    - 관심 등록 수
- **인증/인가**: Spring Security + JWT
- **운영 도구**: 관리자 페이지
    - 도감 관리
    - 출품 관리
    - 주문/배송 관리
    - 분쟁 처리
    - 회원 관리

## 3. 핵심 서비스 구조

### 3.1 카드 도감 중심 구조

- 서비스의 중심은 개별 게시글이 아니라 **카드 도감(CardMaster)** 입니다.
- 카드명, 세트명, 카드번호, 레어리티, 언어, 제조사 등의 정보를 관리합니다.
- 카드 상세 페이지에서 해당 카드의 거래 정보를 집약하여 제공합니다.
- 거래가 활발한 카드부터 우선적으로 관리자가 등록합니다.

### 3.2 출품형 거래 구조

- 회원은 카드 도감에 연결된 자신의 실물 카드를 출품합니다.
- 출품 정보에는 다음이 포함됩니다.
    - 카드 상태(S/A/B/C/D)
    - 감정 여부(PSA/BGS/CGC 등)
    - 실물 이미지
    - 판매 가격
    - 판매 설명
- 구매자는 동일 카드의 여러 출품 중 원하는 상품을 선택하여 구매합니다.

### 3.3 거래 구조

거래는 기본적으로 **판매자와 구매자가 직접 거래**하는 형태로 진행됩니다.

플랫폼은 **주문 관리 / 결제 보관 / 정산 / 분쟁 신고 접수 / 정책 범위 내 중재**를 담당합니다.

### 배송 거래

1. 구매자가 온라인에서 결제합니다.
2. 판매자가 정해진 기간 내 물품을 발송합니다.
3. 판매자가 송장번호를 플랫폼에 입력합니다.
4. 플랫폼은 구매자에게 송장 등록 알림을 발송합니다.
5. 구매자가 상품을 수령합니다.
6. 구매자가 `구매 확정`을 누르면 거래가 완료됩니다.
7. 플랫폼은 수수료를 제외한 금액을 판매자에게 정산합니다.

### 대면 거래

1. 구매자가 결제합니다.
2. 판매자와 구매자가 직접 만나 거래합니다.
3. 구매자가 `거래 완료` 버튼을 누르면 거래가 완료됩니다.
4. 플랫폼은 수수료를 제외한 금액을 판매자에게 정산합니다.

---

## 4. 거래 정책

### 4.1 자동 구매 확정 정책

- 배송완료 후 일정 기간 내 구매자가 구매 확정을 하지 않으면 자동 구매 확정 처리합니다.
- 예시: 배송완료 후 3일(72시간)
- 자동 구매 확정 시 판매자에게 자동 정산됩니다.

### 4.2 판매자 발송 기한 정책

- 판매자는 결제 완료 후 정해진 기한 내 송장번호를 등록해야 합니다.
- 예시: 48시간 또는 72시간 이내
- 기한 내 미발송 시 주문 자동 취소 및 구매자 환불 처리합니다.

### 4.3 분쟁 처리 정책

- 구매자는 상품 상태 불일치, 다른 물건 수령, 배송 파손 등의 경우 분쟁 신고를 할 수 있습니다.
- 분쟁 발생 시 판매자 정산은 즉시 홀딩됩니다.
- 플랫폼은 제출된 증빙 자료를 바탕으로 **운영정책 범위 내에서** 환불 / 반품 / 제재 여부를 검토합니다.
- 플랫폼은 모든 사실관계를 최종 판단하는 기관이 아니며, 필요 시 당사자 간 해결 또는 민형사상 절차 진행을 안내할 수 있습니다.

### 4.4 언박싱 영상 정책

- 배송 거래에서 구매자가 상태 불일치, 오배송, 허위 출품 등의 사유로 분쟁을 제기할 경우,
**언박싱 영상 제출을 필수 정책으로 둘 수 있습니다.**
- 약관에 사전 명시하고, 영상이 없는 경우 분쟁 인정 범위를 제한하는 정책을 검토합니다.

### 4.5 배송비 정책

- 초기에는 **플랫폼 공통 고정 배송비 정책**을 적용합니다.
- 예시: 배송비 3,500원
- 주문 시 자동 부과합니다.
- 판매자가 배송비를 개별 입력하는 방식은 초기에는 적용하지 않습니다.

## 5. 케이스 정리

### Case 1. 정상 거래
- **배송 거래**: 구매자 결제 -> 판매자 발송 -> 송장 등록 -> 구매자 수령 -> 구매 확정 -> 판매자 정산
- **대면 거래**: 구매자 결제 -> 판매자/구매자 대면 거래 -> 구매자 거래 완료 -> 판매자 정산

### Case 2. 판매자가 발송하지 않음
- 정해진 기한 내 송장 미등록 -> 주문 자동 취소 -> 구매자 환불 -> 판매자 패널티 가능

### Case 3. 구매자가 물건을 받았는데 구매 확정을 안 함
- 배송완료 후 일정 시간 경과 -> 자동 구매 확정 -> 판매자 자동 정산

### Case 4. 판매자가 다른 물건을 보냄
- 구매자가 분쟁 제기(`NOT_AS_DESCRIBED` 또는 `FAKE_SUSPECTED`) -> 증빙자료 확인 후 환불 / 반품 / 제재 여부 검토

### Case 5. 카드 상태가 설명과 다름
- 구매자가 분쟁 제기 -> 사진, 설명, 수령품 비교 후 환불, 부분 조정, 판매자 제재 중 검토

### Case 6. 배송 중 파손
- 분쟁 유형: `DAMAGED` -> 포장 책임, 택배 책임 범위 정책 적용하여 환불 또는 택배 클레임.

### Case 7. 구매자가 악의적으로 거짓 분쟁 제기
- 증빙자료 검토 후 반복 시 구매자 제재 가능.

### Case 8. 대면 거래 노쇼
- **판매자 노쇼**: 구매자 신고 -> 주문 취소 및 환불 -> 판매자 패널티
- **구매자 노쇼**: 판매자 신고 -> 정책에 따라 주문 취소 -> 패널티 또는 재예약

## 6. 핵심 도메인 설계

### 6.1 회원/상점
- **Member**: 회원 계정 정보, 권한 및 상태 관리
- **Store**: 공개 프로필, 상점 소개, 판매 목록, 후기 및 평점
- **Address**: 회원 배송지 정보

### 6.2 카드 도감/출품
- **Pokemon**: 포켓몬 메타 데이터 (도감번호, 이름, 속성, 지방 등) 및 CardMaster 연동
- **CardMaster**: 카드 도감 정보
- **CardRequest**: 회원의 카드 추가 요청
- **Listing**: 회원이 등록한 실물 카드 출품 정보
- **ListingImage**: 출품 카드 이미지

### 6.3 주문/거래
- **Order**: 구매 주문
- **Payment**: 결제 및 에스크로 보관
- **Shipment**: 판매자 → 구매자 배송 정보 및 송장 관리
- **Settlement**: 거래 완료 후 판매자 정산 처리

### 6.4 시세/기록
- **TradeHistory**: 최근 거래 이력
- **PriceHistory**: 상태별 시세 데이터
- **Watchlist**: 관심 카드 관리

### 6.5 소통/신뢰
- **InquiryComment**: 상품 문의 댓글
- **Review**: 거래 후 후기 및 평점
- **Dispute**: 분쟁/신고/환불 요청

### 6.6 운영
- **Notification**: 거래/배송/후기/문의 관련 알림
- **AdminActionLog**: 관리자 액션 로그

## 7. 핵심 기능
- **카드 도감**: 조회, 필터링, 최저가, 출품목록, 시세 그래프.
